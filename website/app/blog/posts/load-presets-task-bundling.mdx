# Why Load Presets Matter: Task-Class Bundling for Efficiency

Awesome Arabic Skill has 26 reference files (guides, dialects, validation rules, humanization protocols, etc.).

If you load all 26 for every task, you waste context, tokens, and cost. But if you don't load enough, you miss critical information.

**The solution: Load presets.** Smart bundles optimized for specific tasks.

---

## The Problem

**26 reference files** covering:
- 11 dialects (Masri, Khaliji, Levantine, etc.)
- 12 industry domains (e-commerce, healthcare, finance, etc.)
- Quality standards (audit-mode, humanization, RTL validation)
- Task strategies (seasonal, SEO-AEO, project planning)

**Different tasks need different files:**
- Writing a caption? Need tone, dialect, humanization. Skip audit, skip planning.
- Auditing content? Need audit-mode, RTL checks, dialect-purity. Skip industry guides.
- Planning a campaign? Need seasonal-calendar, strategy, market research. Skip tone guides.

**Loading all 26 every time = massive waste.**

---

## The Solution: Load Presets

A **load preset** is a curated bundle of files for a specific task.

**Example:**

```
Preset "seasonal" = [
  seasonal-calendar.md       (400 lines)
  humanization-protocol.md   (300 lines)
  seo-aeo-masri.md          (250 lines)
]

Total: 950 lines (vs. 25K lines for full load)
```

**Result:** 62% context savings, 75% fewer tokens, 75% cheaper.

## How to Use

### Via CLI

```bash
# Write with seasonal preset
/arabic write caption --preset seasonal

# Audit with full preset
/arabic audit --preset audit-full

# Plan with regional preset
/arabic plan website --preset seo-aeo-gulf

# Use via shell script
bash scripts/load-preset.sh plan
```

### Via Plugin API

```json
{
  "type": "caption",
  "dialect": "masri",
  "brief": "Ramadan greeting",
  "preset": "seasonal"
}
```

## Real Examples

### Scenario 1: Ramadan Campaign (100 Captions)

**Task:** Write 100 Instagram captions for Ramadan  
**Preset:** `seasonal`  
**Why:** Includes seasonal-calendar (Ramadan insights) + humanization + Masri dialect  
**Files loaded:** 3  
**Time per caption:** 1 min  
**Total time:** 100 minutes (~2 hours)  
**Tokens saved:** 75%  
**Cost savings:** 75%

**Without preset (all 26 files):**  
- Time per caption: 3 min  
- Total time: 300 minutes (~5 hours)  
- Cost: $0.08/caption × 100 = $8

**With preset:**  
- Cost: $0.02/caption × 100 = $2  
- **Savings: $6 + 3 hours**

### Scenario 2: Website Audit (50 Pages)

**Task:** Audit 50 landing pages for register, translationese, RTL  
**Preset:** `audit`  
**Files loaded:** 5  
**Time per page:** 2 min  
**Total time:** 100 minutes (~2 hours)  
**Issues caught:** Register mismatches (12), translationese (8), RTL problems (4)

### Scenario 3: Website Planning (Multi-Dialect)

**Task:** Plan 5-page website for Egypt + Gulf markets  
**Presets:** `seo-aeo-masri` + `seo-aeo-gulf` (run twice)  
**Files per run:** 4  
**Turnaround:** 90 minutes vs. 4 hours (full load)

## How to Pick the Right Preset

**Ask yourself:**
1. **What's the task?** Writing? Auditing? Planning?
2. **What's the market?** Egypt? Gulf? Pan-Arab?
3. **What's the content type?** Short-form? Long-form? Holiday-specific?

**Then pick the preset that matches.**
